# Arquitetura

O Cofrinho Real deve crescer com arquitetura simples, organizada e de baixo custo.

## Direção inicial

Durante o MVP:

- Sem backend real.
- Sem banco de dados real.
- Sem integracoes externas.
- Protótipo navegavel e documentado.

## Direção futura

Quando necessário, separar:

- frontend web
- backend
- mobile
- API
- banco de dados
- assets
- testes
- scripts operacionais

## Princípios técnicos

- Comecar simples.
- Evitar microservicos cedo demais.
- Evitar ferramentas caras sem necessidade.
- Documentar decisões importantes.
- Preservar compatibilidade com evolucoes futuras.
