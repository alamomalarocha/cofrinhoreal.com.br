# Checkpoint - Reestrutura multipaginas 001

Data: 2026-07-02

## Objetivo

Registrar a mudança do site Cofrinho Real de uma landing page única e longa para um protótipo visual estático multipaginas.

## Estrutura atual

- `index.html`: Home curta de apresentacao principal.
- `o-que-e.html`: explicação completa do Cofrinho Real.
- `como-funciona.html`: fluxo conceitual de funcionamento.
- `comerciantes.html`: página para comerciantes / Pessoa Jurídica.
- `familias.html`: página para famílias / Pessoa Fisica.
- `educacao.html`: educação financeira com o Pig.
- `jogos.html`: jogos educativos futuros.
- `pig-coins.html`: PigCoins como pontos educativos simbolicos.
- `faq.html`: perguntas frequentes.
- `privacidade.html`, `termos.html`, `cookies.html`, `direitos.html`: páginas legais estáticas.

## Decisões

- A Home deve mostrar apenas o essencial: problema, proposta, PF/PJ, mockup, resumo e CTAs.
- O menu agora aponta para páginas reais, não apenas secoes da Home.
- O projeto continua com `noindex`.
- Header, rodape e identidade visual devem permanecer consistentes.
- Login, busca, App Store/Google Play, Conta PIG e PIX continuam apenas visuais.

## Limites

Esta versao continua sem backend, sem banco de dados, sem login real, sem autenticação real, sem PIX real, sem app real, sem checkout e sem movimentação real de dinheiro.

## Proximos passos sugeridos

- Revisar copy da Home mais curta.
- Revisar consistência visual das páginas internas.
- Evoluir personagens da Vila Pig somente depois da arquitetura visual estar estavel.
