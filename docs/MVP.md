# MVP

O MVP do Cofrinho Real deve validar a ideia com o menor nivel de complexidade possivel.

## Objetivo

Criar uma experiencia navegavel e compreensivel para testar:

- proposta de valor
- identidade visual
- fluxo de troco digital
- papel do Pig
- utilidade para comerciantes
- percepcao de familias e usuarios

## Dentro do MVP

- Prototipo navegavel.
- Home curta com paginas internas estaticas para explicar cada area do projeto.
- Telas simuladas.
- Fluxos sem backend real.
- Conteudo educativo basico.
- Area conceitual do comerciante.
- Materiais A4 simulados.
- Organizacao conceitual da Familia Pig, sem criar imagens ou personagens finais.
- Busca visual estatica, sem backend.
- Modal visual de login, sem autenticacao.
- Paginas legais iniciais para prototipo visual.
- Secoes conceituais de educacao financeira e jogos educativos.

## Fora do MVP

- Backend completo.
- Banco de dados real.
- Integracoes bancarias.
- PIX real.
- Marketplace.
- Chat.
- Rede social.
- Compra ou venda de Pig Coin.
- Criacao completa da Familia Pig.
- Assets oficiais de personagens.
- Login real.
- Busca real.
- App publicado nas lojas.
- Jogos funcionais.
- Coleta real de dados.

## Estrutura visual atual

O MVP deixou de ser uma landing page unica longa. A estrutura atual e um prototipo estatico multipaginas com Home, paginas de explicacao, paginas de publico e paginas legais.

Essa mudanca melhora a clareza sem criar backend, banco de dados, login real, PIX real, app real ou movimentacao real de dinheiro.

## Regra

Se uma funcionalidade nao ajuda a validar a ideia central agora, ela deve esperar.
