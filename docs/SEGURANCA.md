# Seguranca

Seguranca deve ser considerada desde o inicio, mesmo no prototipo.

## Principios

- Coletar o minimo de dados.
- Proteger informacoes de criancas.
- Separar conta comum, infantil e comerciante.
- Registrar eventos importantes.
- Evitar exposicao desnecessaria de saldo.
- Usar autenticacao segura quando houver backend.

## Dados sensiveis

- Nome.
- E-mail.
- Telefone.
- Senha.
- Foto.
- QR Code.
- Numero do cofrinho.
- Historico financeiro.
- Dados de criancas.

## Regras futuras

- Nunca salvar senha em texto puro.
- Nunca exibir dados sensiveis sem necessidade.
- Nunca criar fluxo financeiro real sem auditoria.
- Toda integracao com PIX ou dinheiro real exige revisao de seguranca.
